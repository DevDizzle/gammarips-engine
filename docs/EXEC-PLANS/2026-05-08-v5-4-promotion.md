## Phases

### Phase 1 — Rewrite `signal-notifier/main.py` (1 day)

**Goal:** V5.4-only flow. No V5.3 SQL "top = candidate; break" selection. Fail-closed on V5.4 error. Single email path (operator + subscribers see the same V5.4 content).

**Deliverable changes:**

1. **Candidate pool query (lines ~660-690)**: keep the `LIMIT 10` SQL EXACTLY — V5.3 hard gates (V/OI > 2, moneyness 5-10%, OI/vol floors, spread ≤8%) are upstream of V5.4 and stay. Reframe the comment block: "top-10 candidates V5.4 will rank" rather than "top-10 V5.3 ranker considers".

2. **Earnings overlap exclusion (lines ~743-786)**: keep the `for _, candidate in df.iterrows()` walk that removes earnings-overlap tickers from `df`. The earnings-clean dataframe is what V5.4 sees.
   - **Change:** the `top = candidate; break` selection is REMOVED. Instead, `df` (after earnings filtering) is passed wholesale to signal-ranker.
   - If `len(df) == 0` after earnings filter → fail-closed with `skip_reason="earnings_overlap_all_candidates"` (existing path).

3. **Always call signal-ranker (replace the V5.4 shadow block at lines ~792-805):**
   ```python
   # V5.4 is canonical. No V5.3 SQL fallback. signal-ranker uptime is an SLO.
   report_md = fetch_report_md(target_date)
   ledger_summary = compute_14d_ledger_summary(target_date)
   v5_4_response = call_signal_ranker(df, target_date, entry_day, report_md, ledger_summary)
   if v5_4_response is None:
       # Fail-closed: signal-ranker error → no pick today.
       write_todays_pick_doc(target_date, has_pick=False, skip_reason="v5_4_unavailable")
       post_to_openclaw(format_whatsapp_message(None, target_date, None, has_pick=False, skip_reason="v5_4_unavailable"))
       return True, "V5.4 signal-ranker unavailable. Fail-closed."

   # Find the picked candidate's enriched row for email rendering.
   pick_ticker = v5_4_response["pick"]
   picked_row = df[df["ticker"] == pick_ticker].iloc[0]
   ```

4. **`write_todays_pick_doc` callsite**: pass V5.4's justification + confidence + run_id alongside the picked row. Doc schema gains:
   - `policy_version = "V5_4_AGENT_RANKER"` (was `V5_3_TARGET_80`)
   - `v5_4_run_id` (the ranker run UUID)
   - `v5_4_justification` (Picker prose, 2-3 sentences)
   - `v5_4_confidence` (`"high"` | `"medium"` | `"low"`)
   - `v5_4_runner_up` (ticker)

5. **Email + WhatsApp**:
   - Drop `format_v5_4_email_block` (no longer needed — V5.4 IS the pick).
   - Drop the `operator_html = base_html + format_v5_4_email_block(...)` dual-html split.
   - `format_email_html` (lines ~497-574) takes the picked enriched row as before, AND a new optional `v5_4_meta: dict` arg. Add a Picker-justification block right under the contract card (visible to operator + subscribers — they get the same email).
   - `format_whatsapp_message` similarly — append a one-line "Why: …" trimmed from the V5.4 justification.
   - `fan_out_to_paid_subscribers(subject, html_content)` — pass the same `html_content` operator gets (no more split).

6. **Drop the `todays_v5_4_pick` Firestore writes** in `write_todays_v5_4_pick_doc` — that helper becomes dead code. Either delete it, or keep as a migration artifact and stop calling it. Recommend delete.

7. **`compute_and_write_cohort_stats`** (lines ~577-650): change `policy_version = 'V5_3_TARGET_80'` → `'V5_4_AGENT_RANKER'`. The "official V5.4 pick" cohort is identified by joining ledger to `todays_pick`:
   ```sql
   FROM forward_paper_ledger l
   JOIN (
     SELECT scan_date, ticker FROM `firestore-export.todays_pick_*`  -- or read via Firestore client
     WHERE has_pick = TRUE
   ) p USING (scan_date, ticker)
   WHERE l.policy_version = 'V5_4_AGENT_RANKER'
   ```
   If a Firestore JOIN is awkward in BQ, simpler: maintain a `todays_pick_history` BQ table that signal-notifier appends to on every write. **Recommend** adding that table in Phase 2.

8. **`LIVE_COHORT_START_DATE`** (line 66): `"2026-05-07"` → `"2026-05-08"`.

9. **Email subject + headers**: drop "V5.3 Target 80" copy.
   - `subject = f"GammaRips {entry_day}: {top['ticker']} {top['direction']}"` — already neutral, keep.
   - `format_email_html` line `<p>V5.3 Target 80 · scan {target_date}</p>` → `<p>GammaRips agent ranker · scan {target_date}</p>` (or operator-preferred phrasing).

10. **`STOP_PCT_DISPLAY` / `TARGET_PCT_DISPLAY`** unchanged. V5.4 is a picker change, not a trader change.

**Tests:**
- Unit: `call_signal_ranker` returns None → `run_notifier` returns successfully without sending email + writes empty-state doc.
- Unit: V5.4 returns out-of-set ticker → `picked_row = df[df["ticker"] == pick_ticker].iloc[0]` raises IndexError → caller catches → fail-closed.
- Integration (manual): Cron with stubbed signal-ranker 500 → no email. Cron with healthy signal-ranker → V5.4 pick lands in `todays_pick`.

**Acceptance:** Tomorrow 5/9 07:30 ET cron writes V5.4 pick to `todays_pick`. Webapp banner + x-poster signal + paid subs all reflect V5.4 ticker without any further code change.
